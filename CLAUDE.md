# Echo

Secure video conferencing by Facile Studio. Fork of Jitsi Meet with custom branding and deployment.

## Tech Stack

- **Language**: TypeScript + JavaScript (React 19, React Native 0.79)
- **Bundler**: Webpack 5 (multi-bundle output)
- **Build orchestration**: Makefile
- **Styling**: SCSS (compiled via `sass`, minified via `clean-css`)
- **State management**: Redux (registry-based -- ReducerRegistry, MiddlewareRegistry)
- **Testing**: WebDriverIO (end-to-end, Chrome + Firefox)
- **Linting**: ESLint (`@jitsi/eslint-config`) + TypeScript strict mode
- **Deployment**: Docker (multi-stage: node build, nginx serve) + docker-compose (Prosody, Jicofo, JVB)
- **Core library**: `lib-jitsi-meet` (WebRTC abstraction)
- **Node**: >=24, npm >=11

## Platform File Conventions

The codebase targets both web and React Native. Platform dispatch is file-suffix-based:

- `.web.ts` / `.web.tsx` -- web only
- `.native.ts` / `.native.tsx` -- React Native only
- `.any.ts` / `.any.tsx` -- shared cross-platform
- `.android.ts` / `.ios.ts` -- OS-specific native
- `web/` and `native/` subdirectories within features

TypeScript configs (`tsconfig.web.json`, `tsconfig.native.json`) exclude the opposite platform's files.

## Feature Module Pattern

Every feature under `react/features/<name>/` follows this layout:

- `actionTypes.ts` -- Redux action type constants
- `actions.ts` (or `.any.ts` / `.web.ts` / `.native.ts`) -- action creators
- `reducer.ts` -- Redux reducer (registered via ReducerRegistry)
- `middleware.ts` -- side-effect middleware (registered via MiddlewareRegistry)
- `functions.ts` -- selectors and utility functions
- `constants.ts` -- feature constants
- `components/` -- React components (often split into `web/` and `native/`)

New features register reducers/middleware via the registry, not a central store file.

## Conventions and Gotchas

- All code must pass `npm run lint` (zero warnings) before committing.
- TypeScript strict mode is on. Avoid `any`.
- No `index` files -- import directly from feature files.
- New features must be TypeScript. Convert JS to TS when touching existing files.
- Bundle size limits are enforced in production builds. Justify new dependencies.
- Dev server uses self-signed certs -- browser certificate warnings are expected.
- The Dockerfile uses Node 20 for builds (even though package.json says >=24).
- Docker Compose brings up four services: echo-web (nginx), echo-prosody (XMPP), echo-jicofo (orchestrator), echo-jvb (video bridge).
- `postinstall` runs `patch-package`, `jetify`, and Android autolinking automatically.
- The `libs/` directory is generated and gitignored -- never edit files there.
- `config.js` uses nginx SSI variables for dynamic configuration at runtime.
